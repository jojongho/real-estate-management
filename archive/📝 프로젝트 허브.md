# 📝 부동산 매물 관리 자동화 v2: 프로젝트 허브

> **프로젝트 비전:** 분산된 매물 정보를 중앙화하고, 데이터 수집부터 콘텐츠 발행까지의 전 과정을 자동화하여 업무 효율을 극대화하는 차세대 부동산 중개 솔루션 구축

---

## 1. 🎯 프로젝트 목표

- **안정화**: 기존 매물관리 시스템의 데이터 품질을 95% 이상으로 향상
- **자동화**: PDF, CSV, 웹사이트 등에서 데이터를 자동 수집하여 수동 입력 90% 감소
- **콘텐츠 생성**: 수집된 데이터를 기반으로 일일 브리핑, 마케팅 콘텐츠 자동 생성
- **확장성**: 아파트 외 오피스텔, 상가 등 다른 유형의 매물도 관리할 수 있는 유연한 구조 설계

---

## 2. 🏗️ 시스템 아키텍처

본 프로젝트는 **하이브리드 구조**를 채택하여 코드의 버전 관리는 **Git**에서, 문서와 지식 관리는 **Obsidian**에서 수행합니다.

### **주요 구성 요소:**

- **Google Sheets**: 중앙 데이터베이스 및 UI 역할
  - `매물DB`, `고객DB`, `아파트단지DB` 등 핵심 데이터 저장
  - `등록검색` 시트를 통해 사용자 인터페이스 제공
- **Google Apps Script**: 시트 내 자동화 담당
  - 데이터 검증, ID 자동 생성, 중복 등록 방지 등
- **Python**: 백엔드 데이터 처리 및 수집 담당
  - PDF 파싱, 웹 크롤링, 외부 API 연동
- **Gemini API**: AI 기반 데이터 추출 및 콘텐츠 생성
  - PDF(모집공고) 분석, 마케팅 문구 생성
- **n8n**: 전체 워크플로우 오케스트레이션 (선택 사항)

### **데이터 흐름:**

```
(입력 소스: PDF, 웹, API)
       |
       v
[Python Collector] (데이터 수집/파싱)
       |
       v
[Google Sheets] (정규화된 데이터 저장)
       |
       v
[Apps Script] (시트 내 검증 및 자동화)
       |
       v
[Python Generator] (콘텐츠 생성: 브리핑, 블로그 포스트)
       |
       v
(출력: 이메일, Google Docs, SNS)
```

## 3. 🗺️ 실행 로드맵

### ✅ Phase 0: 현황 분석 및 기획 (완료)
- 기존 시스템 자산 분석 및 문제점 도출
- 통합 마스터플랜 및 기술 스택 확정

### 🚧 Phase 1: 기존 시스템 안정화 (1주)
- **목표**: 데이터 품질 95% 달성 및 사용성 개선
- **주요 과업**:
    - [ ] **Apps Script 개선**: 필수 필드 검증, ID 생성 로직 수정, 중복 방지 강화
    - [ ] **데이터 정제**: 기존 61개 매물 데이터 일괄 정리 스크립트 실행
    - [ ] **사용성 향상**: 고객 상담 이력 자동 기록, Drive 폴더 자동 생성 기능 추가

### 📥 Phase 2: 데이터 수집 자동화 (2주)
- **목표**: 수동 데이터 입력 50% 감소
- **주요 과업**:
    - [ ] **CSV 자동 임포트**: `통합단지DB` CSV 파일들을 Sheets로 자동 동기화
    - [ ] **PDF 파싱**: Gemini Vision API를 활용해 입주자 모집공고문에서 분양가, 옵션 등 자동 추출
    - [ ] **웹 크롤링**: Playwright를 이용해 네이버 부동산 등에서 매물 정보 수집

### 📤 Phase 3: 콘텐츠 발행 자동화 (2주)
- **목표**: 일일 브리핑 및 기본 마케팅 콘텐츠 100% 자동화
- **주요 과업**:
    - [ ] **일일 브리핑 생성**: 신규 등록 매물 정보를 요약하여 매일 아침 Google Docs 문서 생성 및 이메일 발송
    - [ ] **마케팅 문구 생성**: Gemini API를 이용해 매물 특징 기반의 블로그/SNS용 문구 초안 작성

### 🚀 Phase 4: 통합 및 최적화 (1주)
- **목표**: 전체 파이프라인 연결 및 모니터링 대시보드 구축
- **주요 과업**:
    - [ ] **워크플로우 통합**: 데이터 수집부터 콘텐츠 발행까지의 과정을 n8n 또는 Python 스크립트로 연결
    - [ ] **종합 대시보드**: Google Sheets에 단지별/거래유형별/담당자별 성과를 시각화하는 피벗 테이블 및 차트 구성
