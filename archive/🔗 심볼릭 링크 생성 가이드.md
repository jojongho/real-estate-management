# 🔗 심볼릭 링크 생성 가이드

> **목적**: Obsidian Vault와 Git 저장소를 연결하여 통합 작업 환경 구축

## 📍 현재 상태

**Cursor가 생성한 프로젝트:**
- 위치: `D:\Projects\apartment-automation` ✅ (올바른 Git 저장소 위치)
- Git 초기화: 완료 ✅
- 기존 파일 통합: 완료 ✅

**Obsidian 프로젝트 허브:**
- 위치: `D:\Flow System\- Flow\01. Framing\Project\구글 스프레드시트 자동화 매물관리 시스템구축\`
- 심볼릭 링크: ❌ **아직 생성 안 됨**

## 🛠️ 심볼릭 링크 생성 방법

### ✅ 권장: PowerShell (관리자 권한)

```powershell
# 1. PowerShell을 관리자 권한으로 실행 (중요!)
# 시작 메뉴 → PowerShell 우클릭 → "관리자 권한으로 실행"

# 2. 다음 명령어 실행
New-Item -ItemType SymbolicLink `
  -Path "d:\Flow System\- Flow\01. Framing\Project\구글 스프레드시트 자동화 매물관리 시스템구축\code" `
  -Target "d:\Projects\apartment-automation"

# 성공 시 출력:
# Directory: D:\Flow System\...
# Mode    LastWriteTime         Length Name
# ----    -------------         ------ ----
# la---l  2025-10-04            <SYMLNK> code [d:\Projects\apartment-automation]
```

### 대안 1: CMD (관리자 권한)

```cmd
:: 1. CMD를 관리자 권한으로 실행
:: 2. 다음 명령어 실행
mklink /D "d:\Flow System\- Flow\01. Framing\Project\구글 스프레드시트 자동화 매물관리 시스템구축\code" "d:\Projects\apartment-automation"

:: 성공 시 출력:
:: code <<===>> d:\Projects\apartment-automation에 대한 기호화된 링크를 만들었습니다.
```

### 대안 2: 수동 바로가기 (심볼릭 링크 대체)

관리자 권한 없이 작업하려면:

1. `D:\Projects\apartment-automation` 폴더를 우클릭
2. "바로 가기 만들기" 선택
3. 생성된 바로가기를 `D:\Flow System\- Flow\01. Framing\Project\구글 스프레드시트 자동화 매물관리 시스템구축\` 폴더로 이동
4. 이름을 `code`로 변경

> ⚠️ 주의: 바로가기는 심볼릭 링크보다 기능이 제한적입니다.

## ✅ 연결 확인

심볼릭 링크가 제대로 생성되었는지 확인:

```bash
# Git Bash 또는 PowerShell에서
ls -la "d:/Flow System/- Flow/01. Framing/Project/구글 스프레드시트 자동화 매물관리 시스템구축/"

# 심볼릭 링크가 있으면 다음과 같이 표시:
# lrwxrwxrwx  code -> d:/Projects/apartment-automation/
```

또는 Obsidian에서:
- 왼쪽 파일 탐색기에서 `프로젝트 허브.md` 폴더로 이동
- `code/` 폴더가 보이고, 안에 `src/`, `apps-script/` 등이 보이면 성공!
