# 가장 간단한 해결책 - 단지명을 Enum으로 변경! ⚡

**문제**: Ref 타입 Valid_If가 복잡하고 에러 발생

**해결**: 단지명 컬럼을 Text + Enum으로 바꿔버리기!

---

## ✅ 해결 방법 (30초)

### 전제: 이 폼에서만 사용할 거라면

**단지명 컬럼 타입 변경**:

```
1. Data → Tables → 아파트매물 → Columns → 단지명

2. Type 변경:
   ❌ Ref (기존)
   ✅ Text + Enum

3. Type Details:
   Values: (한 줄씩 입력)
   아산배방1단지우방아이유쉘
   아산배방우방아이유쉘2단지

4. Allow other values: ❌ (체크 해제)
5. Input mode: Dropdown

6. Save
```

---

## 🧪 결과

```
이제 이 폼에서 단지명:
✅ 2개만 표시됨
✅ 복잡한 Valid_If 불필요
✅ 에러 없음
```

## ⚠️ 주의사항

### 이렇게 하면 다른 폼에도 영향!

**문제**: 
- 다른 폼에서도 단지명이 2개만 보임
- 전체 단지 리스트 사용 불가

---

## ✅ 해결책: Virtual Column 사용

**정규화 유지하면서 폼별로 다른 값 보여주기**

### Step 1: 원래 단지명 유지

```
단지명 (기존):
Type: Ref → 아파트단지
```

### Step 2: Virtual Column 추가

```
□ Data → Tables → 아파트매물 → Columns → + Add column

□ Column name: 단지명_배방전용

□ Type: Virtual

□ App Formula:
   IF(
     IN([단지명].[단지명], ["아산배방1단지우방아이유쉘", "아산배방우방아이유쉘2단지"]),
     [단지명],
     ""
   )

□ Save
```

### Step 3: 폼에서 가상 칼럼 사용

```
□ 폼 편집
□ 단지명 필드 숨김
□ 단지명_배방전용 필드 추가
□ 이 필드만 표시
```

**문제**: 이 방법도 복잡함 ❌

## 🎯 최종 추천: 별도 뷰에서 Valid_If

**가장 깔끔한 방법**

### 올바른 Valid_If 수식 (ID 사용)

먼저 아파트단지 테이블에서 ID를 확인:

```
□ Data → Tables → 아파트단지
□ 데이터 확인
□ "아산배방1단지우방아이유쉘"의 ID = ?
□ "아산배방우방아이유쉘2단지"의 ID = ?
```

**예시** (ID가 5와 6이면):

```
폼에서 단지명 필드 Valid_If:
IN([단지명].[ID], [5, 6])
```

## 🔍 ID를 모르는 경우

```
□ AppSheet → Data → Tables → 아파트단지
□ 테이블 데이터 보기
□ 해당 단지의 ID 컬럼 확인
```
