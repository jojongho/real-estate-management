# Row Key & Column Order 설정 가이드

---

## 🔑 Row Key 설정

### 현재 화면의 Row Key 옵션

**화면 설명**: "Only edit one specific row, identified by this key."

### ❌ 이 폼에서는 Row Key 설정 불필요!

**이유**:
- 이 폼은 **신규 데이터 등록용**
- 특정 레코드를 수정하는 게 아님
- AppSheet이 자동으로 새 레코드 생성

**결론**: Row Key는 비워두세요! ✅

```
Row key: "" (비움)
또는
Row key: "=" (화면에 그대로 둠)
```

---

### Row Key가 필요한 경우 (참고용)

**다른 폼 예시**:
```
상황: "내 프로필 수정" 폼
목적: 현재 로그인한 사용자의 레코드만 수정

Row key 설정:
[이메일] = USEREMAIL()
또는
[사용자ID] = USERID()

→ 이렇게 하면 해당 사용자의 레코드만 자동으로 로드됨
```

## 📋 Column Order 설정

### 현재 화면의 Column Order

**"Manual" 모드**: 필드 순서와 표시 여부 조정 가능

### ✅ 해야 할 작업

#### 1. 필요한 필드만 체크

**표시할 필드**:
```
✅ 단지명
✅ 동
✅ 호
✅ 타입
✅ 거래유형
✅ 고객 (Ref 필드)
✅ 매매가 (필요시)
```

**숨길 필드**:
```
❌ 관련파일
❌ 분양가 (민감 정보)
❌ 발코니
❌ 옵션비
❌ 프리미엄
❌ 합계
❌ 전세가, 보증금, 월세, 관리비
❌ 매물설명
❌ 입주가능일
❌ 방향
❌ 만기예정일
❌ 접수자, 접수일
❌ 기타 필요 없는 필드들
```

#### 2. 필드 순서 조정

**권장 순서**:
```
1. 단지명
2. 동
3. 호
4. 타입
5. ───── (구분선, 섹션)
6. 거래유형
7. 거래상태
8. ───── (구분선)
9. 고객 (Ref)
10. 매매가 (선택)
```

---

## ⚠️ 중요한 차이점

### Column Order ≠ Valid_If

**Column Order**:
- ✅ **필드 표시 여부** 조정
- ✅ **필드 순서** 조정
- ❌ 드롭다운 **필터링** 불가

**Valid_If**:
- ❌ 필드 표시 여부 조정 불가
- ❌ 필드 순서 조정 불가
- ✅ 드롭다운 **필터링** 가능

---

## 🎯 정확한 작업 순서

### Step 1: Column Order에서 필드 정리

```
□ 필요한 필드만 체크
□ 순서 조정
□ 저장
```

### Step 2: 각 필드 클릭 → Valid_If 설정

**단지명 필드**:
```
□ Column Order에서 단지명 체크
□ 단지명 클릭 (우측 미리보기에서)
□ 아래로 스크롤
□ Valid If 입력:
   IN([단지명].[단지명], ["아산배방1단지우방아이유쉘", "아산배방우방아이유쉘2단지"])
□ Save
```

**다른 필드들도 동일하게**:
```
□ 동, 호, 타입 필드도 필요시 Valid_If 설정
□ 고객 필드는 Ref 설정 확인
```

## 🧪 체크리스트

### Column Order 설정
```
□ 필요한 필드만 체크 ✅
□ 필드 순서 조정 ✅
□ 불필요한 필드 숨김 ✅
```

### Field Level 설정
```
□ 단지명 → Valid_If 설정 ✅
□ 동 → Valid_If 설정 (필요시) ✅
□ 호 → Valid_If 설정 (필요시) ✅
□ 타입 → Valid_If 설정 (필요시) ✅
□ 고객 → Ref 설정 확인 ✅
```

### Row Key
```
□ Row Key 비워두기 ✅
```

## 💡 정리

| 설정 | 위치 | 목적 | 예시 |
|------|------|------|------|
| **Row Key** | View 설정 | 특정 레코드 편집 | `[매물ID] = "ABC123"` |
| **Column Order** | View 설정 | 필드 순서/표시 | 필드 순서 드래그 앤 드롭 |
| **Valid_If** | Field 설정 | 드롭다운 필터링 | `IN([단지명], ["A", "B"])` |
